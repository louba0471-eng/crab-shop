# Web 领域经验

> 搜索、网页抓取、信息获取任务经验

## 搜索（Search）

### Web 搜索
- 使用 `web_search`（Tavily），返回结构化结果和摘要
- `count` 参数：1-20，默认适量
- 支持提取外部内容（需开启 fetch 功能）

### 多搜索引擎
- 技能 `multi-search-engine`：支持 17 个引擎（8 国内 + 9 海外）
- 支持搜索运算符、时间过滤、站点搜索

## 网页抓取（Fetch）

### 基本用法
- `web_fetch`：获取 URL 内容，返回 Markdown 或 Text
- `maxChars` 参数限制最大字符数

### 图片/媒体
- 图片优先使用 `image` 工具直接分析
- 不适合用 web_fetch 获取二进制媒体

---

## 常见模式

### 信息获取流程
1. 搜索了解概况（web_search）
2. 深入阅读特定页面（web_fetch）
3. 需要理解图片内容（image 工具）

### 时效性判断
- 新闻/价格/事件 → 优先使用最新搜索
- 概念/理论/历史 → 可以用缓存内容

---

## 经验教训

1. **搜索结果太多 → 换关键词**，而不是在大量结果中遍历
2. **网页无法访问 → 换引擎/换关键词**，不要死磕
3. **需要最新信息 → 明确加时间限制**，如 "2024年" "最近"
